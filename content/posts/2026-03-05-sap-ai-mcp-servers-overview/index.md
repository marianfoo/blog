If you are searching for **SAP MCP Servers** (Model Context Protocol), you quickly run into the same problem:

Which SAP MCP servers exist (official and community), and what other SAP AI tooling should you know about?

![Screenshot of the sap-ai-mcp-servers README showing the overview tables and categories.](readme.jpg)

That is why I like this new repo:

- **`marianfoo/sap-ai-mcp-servers`**: [SAP MCP Servers and SAP AI Skills list](https://github.com/marianfoo/sap-ai-mcp-servers)

It is a curated list of **SAP MCP servers**, **SAP AI skills**, and **adjacent developer tools**. The README is structured as tables so you can scan by category, compare licenses, and see what is actively maintained.

## What you will find inside

- **Official SAP MCP servers**: for example **SAP Fiori**, **CAP**, **UI5**, and **MDK**.
- **Community SAP MCP servers**: docs, **ABAP and ADT**, integration, Datasphere, **OData**, SAP GUI, HANA.
- **Skills and adjacent tools**: skill repos plus “helper” tooling (plugins, samples, IDE integrations) that often matters just as much as the server itself.

## Why this matters for SAP AI

If you are building an AI assistant for SAP development, MCP servers are the “connectors” that let your agent reliably pull context from documentation and systems. Having a single overview makes it much easier to choose the right building blocks.

## How to contribute

If you know a project that belongs there, you can add it by opening an issue using the [“Add new entry”](https://github.com/marianfoo/sap-ai-mcp-servers/issues/new?template=add-entry.yml) template in the repo.

If you build anything SAP plus MCP related, this is the list I would want to keep open in a browser tab.
